# WayneLock

**Decentralized, trustless password management** — generation, encryption, decentralized storage, Lit-gated keys, and guardian-based recovery on **Filecoin Calibration (FVM)**, without relying on a central password server.

---

## Problem

Today’s biggest security failures don’t come from breaking blockchains—they come from weak Web2 password systems. Passwords are generated by centralized platforms, stored in vulnerable environments, and often reused across services. This creates a single point of failure where breaches, phishing, or credential stuffing can expose emails, apps, and even crypto wallets. Users ultimately lack true ownership, transparency, and control over their credentials.

---

## Solution

WayneLock is a decentralized, trustless password manager that removes reliance on centralized systems by giving users full ownership of password generation, storage, and recovery through cryptographic and distributed systems.

---

## What WayneLock Does

WayneLock generates passwords using **on-chain randomness**, ensuring the process is **provably fair** and **cannot be manipulated** by a centralized service. In practice, this repository combines **verifiable drand beacon randomness** with **device entropy** and transparent client-side derivation (see [diagram 2](docs/diagrams/2-password-generation.mmd)); **gating and recovery** are enforced **on-chain** on Filecoin (FVM) and via **Lit access control conditions**.

These passwords are then secured using **Argon2id-class protection**: **memory-hard, brute-force resistant key stretching**. The **current open-source frontend** implements that layer with **scrypt** (paired with **HKDF-SHA256**) under documented parameters—see [`docs/diagrams/2-password-generation.mmd`](docs/diagrams/2-password-generation.mmd) and `frontend/lib/password-generator.ts`. Migrating the same pipeline to **Argon2id** would further align implementation with the stated design.

Instead of storing passwords in centralized servers, WayneLock **encrypts** and stores password vaults on **decentralized storage**, eliminating single points of failure. This codebase uploads ciphertext through **Filecoin Warm Storage / Synapse** and records **metadata** on-chain.

Access to the vault is controlled through **on-chain smart contracts**, which define clear, transparent conditions for when and how credentials can be accessed.

To ensure no single entity has control, WayneLock uses **Lit Protocol** for **distributed key management**, splitting the encryption key across multiple nodes and only reconstructing it when **predefined conditions** are met (e.g. guardian threshold satisfied on the FVM contract).

It also enables **secure recovery** mechanisms (such as **threshold-based access** or **guardians**), removing the need for insecure password reset flows.

## WayneLock project overview (this repository)

### Monorepo layout

- **`frontend/`** — Next.js 16 app (App Router): landing page, **Create** (`/create`), **Vault** (`/vault`), **Owner recovery** (`/recovery/owner`), **Guardian approval** (`/recovery/guardian`). Uses **Wagmi**, **RainbowKit**, **viem**, **Lit Protocol** client, **@filoz/synapse-core** for Filecoin Pay / uploads, and client-side crypto in `lib/`.

### Main user flows

1. **Create / add entry** — Generate a password locally; encrypt for Lit; optionally fund **Filecoin Pay (USDFC)**; upload encrypted blob via Synapse / FWSS; register vault and/or **addEntry** on the guardian contract with JSON metadata (piece CID, service URL, Lit ciphertext + hash).

2. **Vault** — List entry UIDs from the contract; fetch blob from storage; decrypt DEK via Lit when ACCs pass; decrypt password locally.

3. **Recovery** — Owner calls **`startRecovery`**; guardians call **`approveRecovery`** until threshold; Lit sees **`isRecoveryApprovedForOwner`**; owner can decrypt with the same Lit + local flow.

### Key libraries (frontend)

| Area | Role |
|------|------|
| `lib/password-generator.ts` | drand + device secret + HKDF + scrypt pipeline. |
| `lib/password-encrypt.ts` | Encrypt password payload for Lit / local testing. |
| `lib/lit-recovery.ts` | Lit encrypt/decrypt of DEK; ACC tied to guardian contract + chain. |
| `lib/fwss.ts` / `lib/synapse.ts` | Warm Storage upload and Synapse wallet client. |
| `lib/guardian-recovery-contract.ts` | ABI + helpers for the WayneLock guardian/vault contract. |
| `lib/filecoin-calibration.ts` | Chain definition **314159** (Filecoin Calibration). |

### External systems

- **Filecoin Calibration** — FVM testnet for smart contracts and USDFC / Filecoin Pay (see contracts below).  
- **drand** — Public randomness beacon (verified rounds) for the password seed pipeline.  
- **Lit Protocol** — Threshold cryptography and ACCs; key material gated by on-chain recovery state.  
- **Filecoin Warm Storage (Synapse)** — Durable piece storage referenced from on-chain metadata.
